# HDF5 Extraction, Validation, and Merge Pipeline Guide

This document describes how to validate, repair, and merge HDF5 files
generated by:

-   `extract_event_images_threaded.py`
-   `make_hdf5s_from_baseline_csv.py`
-   `merge_type_shards.py`

------------------------------------------------------------------------

## 🔒 Important Operational Rules

-   Never run extractor and merge jobs at the same time.

-   Always disable HDF5 file locking on network storage:

    ``` bash
    export HDF5_USE_FILE_LOCKING=FALSE
    ```

-   Always validate shards before merging.

-   Always validate the final file after merging.

------------------------------------------------------------------------

# 1️⃣ corruption_validator.py

### Purpose

Checks: - Image count - Manifest alignment - Missing indices - Orphan
indices - Duplicate image hashes

### Usage

``` bash
python corruption_validator.py path/to/shard.hdf5
```

### When To Run

-   After extraction completes
-   Before merge
-   After merge
-   After crashes or resume events

------------------------------------------------------------------------

# 2️⃣ diagnostic_uniqueness.py

### Purpose

Computes SHA1 hashes of images to detect exact pixel duplicates.

### Usage

``` bash
python diagnostic_uniqueness.py shard.hdf5
```

Optional:

``` bash
--dump-duplicates duplicates.csv
--list-sample 5
```

### Important

Duplicates do NOT automatically indicate corruption. Duplicates may be
biologically legitimate.

------------------------------------------------------------------------

# 3️⃣ inspect_duplicates.py

### Purpose

Visually inspect duplicate image groups.

### Usage

``` bash
python inspect_duplicates.py shard.hdf5 --max-groups 5 --out-dir dup_imgs
```

### What It Does

-   Prints slide_id/frame_id/x/y
-   Saves PNG thumbnails of duplicate crops
-   Lets you visually confirm duplicates

------------------------------------------------------------------------

# 4️⃣ truncate_resume_error.py

### Purpose

Fix resume duplication where images were appended twice but manifest
only references half.

### When To Use

Only when corruption_validator reports symmetric missing indices
(exactly half).

------------------------------------------------------------------------

# 5️⃣ repair_hdf5_from_manifest.py

### Purpose

Reconstruct `/features` table from manifest.

### Usage

``` bash
python repair_hdf5_from_manifest.py shard.hdf5
```

------------------------------------------------------------------------

# 6️⃣ reconstruct_manifests_from_shards.py

### Purpose

Rebuild `.manifest.parquet` from original per-shard CSV.

### Usage

``` bash
python reconstruct_manifests_from_shards.py \
  --type-dir data/common_cell \
  --csv-root data/_tmp_csvs
```

------------------------------------------------------------------------

# 7️⃣ merge_type_shards.py

### Purpose

Stream-merge shard HDF5 files into a final HDF5.

### Usage

``` bash
export HDF5_USE_FILE_LOCKING=FALSE
python merge_type_shards.py data/common_cell --reset
```

### Before Running

-   Ensure no extractor jobs running
-   Validate shards using corruption_validator
-   Remove existing final file if resetting

------------------------------------------------------------------------

# 📊 Recommended Validation Workflow

After extraction completes:

``` bash
for shard in data/common_cell/*.hdf5
do
  python corruption_validator.py $shard
done
```

If any shard reports missing indices: - Inspect - Truncate or
regenerate - Repair features

Then:

``` bash
python merge_type_shards.py data/common_cell --reset
python corruption_validator.py data/common_cell/common_cell.final.hdf5
```

------------------------------------------------------------------------

# 🧠 Healthy Dataset Criteria

Healthy shard: - Images == Included rows - Missing indices == 0 -
Duplicate groups may exist (normal)

Healthy final file: - Images == feature rows - No missing indices

------------------------------------------------------------------------

This guide ensures robust validation and safe merging of HDF5 extraction
pipelines.
